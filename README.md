# Tarun Travel Hub

Client website for Tarun Travel Hub, built with Next.js App Router and React.

## Commands

```bash
npm install
npm run dev
npm run lint
npm run build
npm start
```

## Environment

Copy `.env.example` to `.env.local` and set the production website and
official profile URLs:

```env
NEXT_PUBLIC_SITE_URL=https://your-domain.com
NEXT_PUBLIC_GOOGLE_BUSINESS_PROFILE_URL=https://g.page/r/your-business-profile
NEXT_PUBLIC_GOOGLE_REVIEW_URL=https://g.page/r/your-business-profile/review
NEXT_PUBLIC_GOOGLE_MAPS_EMBED_URL=https://www.google.com/maps/embed?pb=your-embed-code
NEXT_PUBLIC_GOOGLE_MAPS_DIRECTIONS_URL=https://www.google.com/maps/dir/?api=1&destination=your-business-location
GOOGLE_SITE_VERIFICATION=your-google-verification-code
```

## Project Structure

```text
app/                       Route pages, metadata and global styles
  cabs/[slug]/             Static cab and fare detail pages
components/
  cabs/                    Fleet cards and grids
  forms/                   Interactive enquiry forms
  layout/                  Header, footer and global WhatsApp button
  sections/                Reusable page sections
  ui/                      Small shared presentation components
data/                      Client-editable business and website content
lib/                       Shared URL and integration helpers
public/                    Local static assets
```

## Content Management

- Business phone, email, address and social links: `data/site.js`
- Cab fleet, fares and popular routes: `data/cabs.js`
- Services, reviews and gallery content: `data/content.js`
- Global design styles: `app/globals.css`

Cab pages are generated from each vehicle `slug`, for example:

```text
/cabs/innova-crysta
```

## Deployment Checklist

1. Replace placeholder contact details and social URLs in `data/site.js`.
2. Set `NEXT_PUBLIC_SITE_URL` to the final domain.
3. Confirm airport, local and outstation fares with the client.
4. Replace placeholder Google Maps location if required.
5. Keep the business name, address, phone, category and opening hours in
   `data/site.js` identical to the Google Business Profile.
6. Add the production website URL to the Google Business Profile.
7. Run `npm run lint` and `npm run build`.

## Genuine Google Review Process

Use the Google Business Profile's official "Ask for reviews" link as
`NEXT_PUBLIC_GOOGLE_REVIEW_URL`. Request reviews only from customers who
completed a real trip or service.

1. Send one polite request shortly after the trip is completed.
2. Ask for honest feedback without requesting a specific star rating.
3. Send at most one gentle reminder if the customer does not respond.
4. Do not offer discounts, gifts or payment in exchange for reviews.
5. Do not filter requests so that only satisfied customers receive them.
6. Respond professionally to both positive and negative reviews.
7. Use customer-approved testimonials only in `data/content.js`.

A target such as 50 reviews is an outreach goal, not a guarantee. Every
published review must be voluntary and written by a real customer.

## Google Indexing

After deployment:

1. Set `NEXT_PUBLIC_SITE_URL` to the final HTTPS domain before building.
2. Verify the domain in Google Search Console using
   `GOOGLE_SITE_VERIFICATION`.
3. Submit `https://your-domain.com/sitemap.xml` in Search Console.
4. Request indexing for the home page and important cab pages.
5. Confirm `https://your-domain.com/robots.txt` is publicly accessible.

Canonical URLs, page metadata, Open Graph tags, Twitter cards, robots
directives and the XML sitemap are generated by the application.
